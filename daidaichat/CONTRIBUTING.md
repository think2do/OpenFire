# 贡献指南

感谢你考虑为 DaiDaiChat 做出贡献！

## 如何贡献

### 报告 Bug

如果你发现了 Bug，请创建一个 Issue 并包含以下信息：

- **Bug 描述**：清晰简洁地描述问题
- **重现步骤**：详细列出重现步骤
- **期望行为**：描述你期望发生什么
- **实际行为**：描述实际发生了什么
- **环境信息**：
  - iOS 版本
  - Xcode 版本
  - 设备型号

### 提出新功能

如果你有新功能建议，请创建一个 Issue 并包含：

- **功能描述**：清晰描述你建议的功能
- **使用场景**：说明这个功能的使用场景
- **可选实现方式**：如果有想法，可以提出实现建议

### 提交代码

1. Fork 本仓库
2. 创建你的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交你的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启一个 Pull Request

### 代码规范

- 遵循 Swift 官方代码风格指南
- 为公共 API 添加文档注释
- 保持代码简洁易读
- 添加必要的注释说明复杂逻辑

### Commit 信息规范

使用清晰的 commit 信息：

- `feat:` 新功能
- `fix:` Bug 修复
- `docs:` 文档更新
- `style:` 代码格式调整
- `refactor:` 代码重构
- `test:` 测试相关
- `chore:` 构建或辅助工具变动

示例：
```
feat: 添加消息搜索功能
fix: 修复对话顺序错乱问题
docs: 更新 README 安装说明
```

## 行为准则

- 尊重所有贡献者
- 接受建设性的批评
- 专注于对社区最有利的事情
- 对其他社区成员表示同理心

## 问题？

如有任何问题，欢迎通过 Issue 提问。

感谢你的贡献！ 🎉
